# ZEON — Описание платформы и User Cases

## 1. Общее описание

**ZEON** — финтех-платформа для автоматизированного инвестирования в криптовалюты. Пользователи выбирают торговую стратегию, вносят капитал, а платформа автоматически торгует с помощью алгоритмических ботов. Платформа работает на стеке React/TypeScript (фронтенд), Express/Node.js (бэкенд), PostgreSQL (БД).

**Ключевое ценностное предложение:** «Деньги работают — вы отдыхаете». Пользователь выбирает уровень риска, вносит USDT и получает доходность от 1.8% до 12% в месяц через автоматический трейдинг.

## 2. Архитектура и модули

### 2.1 Торговый движок (Engine)
- **14 алгоритмических стратегий** для 7 криптовалют (BTC, ETH, BNB, SOL, XRP, DOGE, ADA)
- Стратегии: squeeze breakout, leveraged momentum, EMA revert, grid scalping, trend pullback, range harvest, volume burst, spike catcher, Keltner revert и др.
- **Walk-Forward фильтр** — блокирует сделки, если скользящая вероятность win-rate или EV ниже порога
- Расчёт проскальзывания (slippageBps), комиссий (feesBps), автоматическое управление ордерами
- **Engine Events:** TICK_OK/TICK_FAIL, TRADE_OPEN/TRADE_CLOSE, DD_TRIGGER, daily_pnl, settlement

### 2.2 Система инвестиций
- **3 риск-тира:** LOW (Стабильный, 1.8–3.6%/мес), CORE (Активный, 3–6.5%/мес), HIGH (Агрессивный, 5–12%/мес)
- **Vault-система:** principal (вложено), profit (накопленная прибыль), taxes (резерв налогов)
- **Auto-Sweep:** автоматическое распределение процента прибыли между хранилищами
- **Машина состояний:** NOT_INVESTED → INVESTED_ACTIVE → WITHDRAWING

### 2.3 Финансовые операции
- **Пополнение:** USDT (TRC20), банковская карта (RUB → USDT конвертация)
- **Инвестирование:** выбор стратегии, ввод суммы, подтверждение
- **Вывод:** 4-eyes workflow (PENDING_REVIEW → PENDING_APPROVAL → APPROVED)
- **Начисление прибыли:** автоматическое, отражается в позициях и equity-снимках
- **Все операции логируются** в таблице operations с полной историей

## 3. Пользовательские роли

### 3.1 Неавторизованный пользователь
- Видит лендинг с описанием платформы
- Может пройти демо-флоу
- Может зарегистрироваться/авторизоваться

### 3.2 Авторизованный пользователь (Инвестор)
- Полный доступ к платформе после прохождения онбординга и KYC

### 3.3 Демо-пользователь
- Получает сессию с предзаполненными данными (10,000 USDT, позиции, история)
- Полный доступ к интерфейсу без реальных денег

### 3.4 Администратор
- Управление пользователями, KYC, выводами, инцидентами
- Мониторинг движка, аудит-логи

### UC-02: Демо-флоу (Пробный прогон платформы)
**Актор:** Потенциальный пользователь
**Цель:** Оценить платформу без регистрации и вложений
**Сценарий:**
1. Нажимает «Демо» на лендинге → переход на /demo/register
2. **Регистрация:** вводит телефон и email, подтверждает OTP-кодом (1234)
3. **Анкета (Risk Profiling):** 4 вопроса — опыт, реакция на просадку 20%, горизонт, доля портфеля
4. **Рекомендация стратегии:** система рассчитывает балл и рекомендует тир:
   - Балл ≤ 4 → Стабильный
   - Балл ≤ 8 → Активный
   - Балл > 8 → Агрессивный
   - Пользователь может выбрать другую
5. **Способ пополнения:** выбор между картой и криптовалютой
6. **Пополнение:** ввод суммы на цифровой клавиатуре, подтверждение
7. **Живой портфель:** видит реалтайм-симуляцию роста баланса (тики каждые 60мс = 30 мин реального времени)
8. **Конверсия:** нажимает «Открыть портфель» → создание реальной демо-сессии → попадает на настоящий экран Портфеля с данными
**Альтернативный путь:** Кнопка «Демо-вход (с данными)» на лендинге — мгновенный вход с предзаполненными данными, без прохождения анкеты

### UC-03: Регистрация и онбординг
**Актор:** Новый пользователь
**Цель:** Создать аккаунт и пройти верификацию
**Сценарий:**
1. Нажимает «Войти» → авторизация через Replit OIDC
2. **Верификация контакта:** получает код, вводит для подтверждения
3. **Правовое согласие:** принимает Условия использования и Политику конфиденциальности (версионируется, с таймстемпом)
4. **KYC (Верификация личности):**
   - Статусы: NOT_STARTED → IN_REVIEW → APPROVED / NEEDS_ACTION / REJECTED
   - В демо: автоматическое одобрение через 2 секунды
   - В продакшене: попадает в очередь на ревью администратора
5. **Smart Start (опционально):** анкета для определения риск-профиля и рекомендации стратегии
6. **ДУ-декларация (опционально):** подписание декларации доверительного управления
7. Попадает на главный экран Портфеля

### UC-05: Выбор и инвестирование в стратегию
**Актор:** Авторизованный пользователь
**Цель:** Вложить средства в алгоритмическую стратегию
**Сценарий:**
1. Переходит в «Стратегии» (/strategies)
2. Видит 3 карточки стратегий с сегментным переключателем (Стабильный / Активный / Агрессивный)
3. Для каждой стратегии видит:
   - Название, эмодзи, тегировка
   - Доходность (мин–макс % в месяц)
   - Sparkline-график роста
   - Калькулятор: выбирает сумму (500/1K/5K/10K) → видит прогноз на 3/6/12 месяцев
4. Нажимает «Начать зарабатывать»
5. **Инвестиционный шит:**
   - Выбор уровня риска (LOW / CORE / HIGH)
   - Видит ожидаемую доходность и макс. просадку
   - Вводит сумму (доступный баланс показан)
   - Нажимает «Продолжить»
6. **Подтверждение:**
   - Видит сумму, комиссию, баланс до/после
   - Нажимает «Инвестировать»
7. **Результат:** «Инвестиция выполнена» → кнопка «Готово» → возврат на Портфель

### UC-08: Вывод средств
**Актор:** Авторизованный пользователь
**Цель:** Вывести средства с платформы
**Сценарий:**
1. Переходит на /withdraw
2. Вводит сумму вывода
3. Указывает адрес (или выбирает из белого списка)
4. Видит комиссию сети и итоговую сумму

8. Средства отправляются, пользователь получает уведомление
9. После успеха → возврат на Портфель

### UC-12: Настройки безопасности
**Актор:** Авторизованный пользователь
**Цель:** Защитить аккаунт
**Сценарий:**

**A. Двухфакторная аутентификация (2FA):**
1. Переходит в «Настройки» → «Безопасность»
2. Включает 2FA: сканирует QR-код в приложении (TOTP)
3. Вводит 6-значный код для подтверждения
4. 2FA запрашивается при: выводах, добавлении адресов, отключении 2FA

**B. Белый список адресов:**
1. Включает функцию
2. Добавляет TRC20 адреса (с подтверждением 2FA)
3. Новые адреса имеют задержку активации (addressDelay, напр. 24 часа)
4. Выводы возможны ТОЛЬКО на адреса из белого списка

**C. Анти-фишинг код:**
1. Устанавливает уникальную строку
2. Этот код включается во все системные email-уведомления
3. Позволяет отличить реальные письма от фишинга
