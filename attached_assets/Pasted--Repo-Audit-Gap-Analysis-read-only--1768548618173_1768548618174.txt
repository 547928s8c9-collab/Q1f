Сделай “Repo Audit / Gap Analysis” и НИЧЕГО НЕ МЕНЯЙ в коде (read-only режим).
Цель: понять, что уже реализовано в проекте, а что нет, по нашему плану фич.

=== Наш общий план фич (чеклист) ===
1) Inbox/Notifications: лента событий по деньгам → карточки со статусом и next step (Revolut-like)
2) Statements / Export: месячная выписка + CSV операций + отчёт по fee и P&L
3) Goals на Vaults: цель + прогресс + авто-отчисление процента профита
4) Risk controls для стратегии: Pause, лимит просадки, автопауза при DD > X
5) Status page внутри приложения: сеть перегружена / выплаты задерживаются / плановые работы
6) Smart Start onboarding: 3 вопроса → рекомендованный набор стратегий и лимит суммы

=== Что нужно сделать ===
A) Repo Snapshot (1–2 экрана текста):
- Стек (frontend/backend), где роутер, где API, где DB слой
- Список UI страниц/маршрутов (routes) которые реально существуют
- Список API endpoints (routes) которые реально существуют
- Список таблиц/моделей в БД (Drizzle schema)

B) Feature Checklist Table:
Сделай таблицу (markdown) с колонками:
- Feature (из списка 1–6)
- Status: DONE / PARTIAL / NOT STARTED
- Evidence: конкретные ссылки на файлы/папки + routes/endpoints + таблицы/поля
- How to verify: точные шаги (URL, клики, какой результат ожидается)
- Gaps: чего не хватает до “DONE”

C) “Что уже сделано сверх плана / фундамент”
Отдельно перечисли базовую инфраструктуру, если она есть (например: Postgres + Drizzle миграция, /api/bootstrap, Operation log, seed данные и т.п.) — тоже с evidence.

D) Команды проверки
Запусти только безопасные команды чтения/запуска (без миграций/дропов):
- как минимум: установка зависимостей (если нужно), запуск dev, и 2–3 быстрых проверки UI/эндпоинтов
В отчёте укажи “Commands run” и результаты кратко.

E) В конце дай “Recommended next”
- 3 самых логичных следующих итерации (в порядке приоритета)
- каждая итерация: цель (1 строка) + почему (1 строка)

Важно:
- Не делай рефакторинг, не форматируй файлы, не меняй код.
- Если что-то невозможно проверить (например, нет данных/логина) — опиши, что мешает, и предложи обходной способ проверки.
